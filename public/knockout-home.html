It means the author of the page have made it in a way that allows you to distribute interface elements over multiple devices. Mobile ones included! Check some wicked stuff!

How is that possible?

The concept is quite simple: if humans can exchange messages in a chat room, why couldn't websites? In practice, we've built a special kind of Node.js server from which you can request a channel and name some participants. From that point, it's up to the author define what the pieces of his UI will chat about ;) Check out proto-documentation here. Or by doing view-source on a demo.

Can I use it in my website or app?

Sure! The project was born during the Node Knockout event (a 48-hour-do-what-you-want/can-competition) so expect some unpolished/untested stuff. Aware of that, tell your site/app to follow the step below and you should be good to go:

  1. Access the request channel page
  2. Tell it how many UI elements will participate in the channel
  3. Tell them who should they listen to
  4. Teach them to understand each other

Let us hope community will help each other in the forums.
